        Examples.ENode (Anoma v0.17.0)

        I give examples of the ENode.

        Currently I make networks in Phase_1 of the example. Meaning that network configurations I provide, are not cloned. The given storage names will be written over for any shared set of storage.

        When phase_2 of the changes come in, the signature and examples of this module in particular will likely change quite a bit to return persistent networks that other examples can use non-destructively. And thus every example can simply add on to another. However since we don't have this examples, are expected to clean out the state before running.
